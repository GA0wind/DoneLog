# DoneLog - 日常事件管理应用

DoneLog 是一个基于 uni-app x 开发的日常事件管理应用，帮助用户记录和管理日常事件的完成时间与下次计划时间。

## 功能特性

### 事件管理
- ✅ 新增事件：填写事件名称、选择分类标签、设置间隔天数或下次完成时间
- ✅ 编辑事件：修改事件的所有字段，或直接结束事件
- ✅ 删除事件：滑动删除或点击删除按钮
- ✅ 标记完成：自动更新完成时间，支持选择下次完成时间

### 标签系统
- ✅ 自定义标签（如"运动"、"学习"、"工作"、"喂猫"等）
- ✅ 标签支持增、删、改
- ✅ 每个事件可关联多个标签
- ✅ 标签可作为筛选条件

### 智能排序与展示
- ✅ 首页以卡片列表展示所有未完成事件
- ✅ 智能排序：今日应完成的事件优先，其余按距今天数排序
- ✅ 过期任务红色标记提示
- ✅ 支持搜索和标签筛选

### 历史记录
- ✅ 查看某个事件的历史完成记录（时间线）

## 技术栈

- **框架**: uni-app x (UTS)
- **数据存储**: uni.setStorageSync (本地存储)
- **UI组件**: 原生组件

## 数据模型

### 事件对象 (Event)
```typescript
{
  id: number,              // 主键
  name: string,            // 事件名称
  labelIdList: string,     // 关联的标签id列表（逗号分隔）
  intervalDays: number,    // 间隔天数（和下次完成时间二选一）
  nextDueDate: number,     // 自定义下次完成时间（时间戳）
  createTime: number,      // 创建时间（时间戳）
  updateTime: number,      // 更新时间（时间戳）
}
```

### 事件完成记录 (CompletionRecord)
```typescript
{
  id: number,              // 主键
  eventId: number,         // 事件id
  lastCompleteDate: number,// 上次完成时间（时间戳）
  createTime: number,      // 创建时间（时间戳）
}
```

### 标签对象 (Label)
```typescript
{
  id: number,              // 主键
  name: string,            // 标签名称
  color: string,           // 标签颜色
  sortOrder: number,       // 排序
}
```

## 使用说明

### 1. 安装依赖
确保已安装 HBuilderX 并配置好 uni-app x 开发环境。

### 2. 运行项目
在 HBuilderX 中打开项目，点击运行按钮即可运行到模拟器或真机。

### 3. 功能使用

#### 添加事件
1. 点击首页右下角的"+"按钮
2. 填写事件名称
3. 选择标签（可选）
4. 选择重复方式：
   - 按间隔天数：设置每隔多少天完成一次
   - 自定义下次完成时间：选择具体的下次完成日期
5. 点击保存

#### 标记完成
1. 在事件列表中找到要完成的事件
2. 点击"完成"按钮
3. 如果是自定义下次完成时间的事件，可以选择新的下次完成时间或取消

#### 管理标签
1. 点击底部标签栏的"标签"标签
2. 点击右下角的"+"按钮添加新标签
3. 点击标签可以编辑标签名称和颜色
4. 点击"删除"按钮可以删除标签

#### 查看历史记录
1. 在事件列表中点击事件卡片
2. 进入详情页后可以看到该事件的所有完成记录

## 开发说明

### 添加新功能
1. 在对应的 utils 文件中添加数据操作方法
2. 在页面中调用相应的方法
3. 更新 pages.json 添加新页面路由（如需要）

### 修改样式
所有页面的样式都写在各自页面的 `<style>` 标签中，可以直接修改。

### 数据持久化
所有数据都存储在本地，使用 `uni.setStorageSync` 和 `uni.getStorageSync` 进行读写。

## 注意事项

1. 本项目使用 uni-app x 和 UTS 语言开发，需要 HBuilderX 3.8.0 或更高版本
2. 数据存储在本地，清除应用数据会丢失所有记录
3. 建议定期备份重要数据
